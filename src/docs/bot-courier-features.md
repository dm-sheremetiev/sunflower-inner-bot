# 📹 Відеофіксація доставки та 🚗 Зміна водія

Цей документ описує роботу з двома ключовими функціями Telegram-бота для кур'єрів: відеофіксація доставки букетів та керування змінами водіїв через платформу SunWays.

---

## 1. Відеофіксація доставки букетів

### Призначення

Кур'єр фіксує факт доставки замовлення відеоповідомленням. Бот автоматично:

- пересилає відео у спільну групу фіксації;
- змінює статус замовлення в CRM на "Доставлено";
- повідомляє менеджера про результат.

### Як користуватися

1. Запишіть відео доставки.
2. Відправте його боту із підписом у форматі:

```
Доставка 12345
```

де `12345` — номер замовлення з CRM.

3. Бот перевірить:
   - чи існує таке замовлення;
   - чи є ви кур'єром у системі;
   - чи призначені відповідальні на замовлення.

4. Якщо все гаразд:
   - відео пересилається у групу відеофіксації;
   - статус замовлення змінюється на "Доставлено";
   - кур'єр отримує підтвердження;
   - менеджер отримує повідомлення про доставку.

### Можливі помилки

| Ситуація | Повідомлення бота |
|---|---|
| Неправильний формат | `Суворий формат тексту: Доставка 0000, де 0000 це номер замовлення` |
| Замовлення не знайдено | `Такого замовлення не існує, перевірте будь ласка номер замовлення та спробуйте ще раз.` |
| Користувач не є кур'єром | `Вибачте, цей функціонал доступний тільки кур'єрам.` |
| Немає відповідальних | `На це замовлення спершу треба призначити відповідальних.` |
| Помилка пересилки | `Сталася якась помилка при пересилці відео у групу. Спробуйте це зробити власноруч.` |

### Хто має доступ

Тільки користувачі, чиї Telegram username додані до списку кур'єрів (змінна середовища `COURIERS`).

## 2. Прикріплення фото збірки / пакування

### Призначення

Флористи та відповідальні прикріплюють фото збірки (без пакування) та фото у пакуванні до замовлення. Бот автоматично надсилає фото клієнту в чат через CRM та змінює статус замовлення.

### Як користуватися

1. Перейдіть до списку замовлень командою `/my_orders` або `/order`.
2. Оберіть замовлення зі списку.
3. Під деталями замовлення з'являться inline-кнопки:
   - **Прикріпити фото збірки** — фото без пакування.
   - **Прикріпити фото з пакуванням** — фінальне фото.
4. Надішліть одне фото.
5. Підтвердьте дію кнопкою **Надіслати це фото та змінити статус** або скасуйте кнопкою **Відміна**.
6. Бот:
   - змінює статус замовлення;
   - надсилає фото клієнту в чат із повідомленням "Надсилаємо фото збірки/пакування на затвердження✨".

## 3. Зміна водія (SunWays)

### Призначення

Кур'єр керує своєю робочою зміною: починає, завершує або скасовує її. Дані передаються на платформу SunWays для обліку робочого часу, пробігу та прив'язки авто.

### Як почати

Введіть у чат з ботом:

```
зміна водія
```

Бот покаже три inline-кнопки:

- **Почати зміну**
- **Завершити зміну**
- **Скасувати сьогоднішню зміну**

### Почати зміну

Крок за кроком:

1. Натисніть **Почати зміну**.
2. Бот завантажить список доступних авто із SunWays і покаже їх inline-кнопками (наприклад: `KA6370PX (Fiat Doblo)`).
3. Оберіть авто.
4. Введіть стартовий пробіг (лише число, в км). Наприклад: `15230`.
5. Надішліть геолокацію кнопкою **Надіслати геолокацію**.
6. Бот покаже зведення (авто, координати) та кнопку **Почати зміну** для підтвердження.
7. Після підтвердження бот надсилає дані на SunWays і відповідає:

```
Зміну розпочато успішно. Планове завершення: 25.03.2026, 21:00
```

#### Що передається на SunWays при старті

| Поле | Опис |
|---|---|
| `telegramUsername` | Username кур'єра |
| `vehicleId` | ID обраного авто |
| `odometerStart` | Стартовий пробіг |
| `lat`, `lng` | Координати геолокації |

### Завершити зміну

1. Натисніть **Завершити зміну**.
2. Введіть фінальний пробіг (лише число, в км). Наприклад: `15280`.
3. Підтвердьте кнопкою **Завершити зміну**.
4. Бот надсилає на SunWays поточний час (ISO, таймзона Київ) як `endTime` та `odometerEnd`.
5. Відповідь: `Зміну завершено.`

#### Що передається на SunWays при завершенні

| Поле | Опис |
|---|---|
| `telegramUsername` | Username кур'єра |
| `endTime` | Поточний час у форматі ISO (Europe/Kyiv) |
| `odometerEnd` | Фінальний пробіг |

### Скасувати сьогоднішню зміну

1. Натисніть **Скасувати сьогоднішню зміну**.
2. Бот надсилає `DELETE` запит на SunWays.
3. Сервер видаляє активну зміну кур'єра (разом з подіями).
4. Відповідь: `Сьогоднішю зміну скасовано.`

### Можливі помилки

| Ситуація | Повідомлення бота |
|---|---|
| Кур'єра не знайдено | `Кур'єра не знайдено за telegram username.` |
| Зміну вже розпочато | `Зміну вже розпочато.` |
| Авто зайняте | `Авто зайняте іншою активною зміною.` |
| Активна зміна не знайдена | `Активну зміну не знайдено.` |
| Помилка авторизації | `Помилка авторизації Sunways API.` |
| Сесія застаріла (10 хв) | `Сесію не знайдено або вона застаріла. Почніть заново.` |

### Хто має доступ

Тільки кур'єри зі списку `COURIERS`.

### Автозакриття змін

Щодня о **02:00 за київським часом** бот автоматично надсилає запит на SunWays для закриття незакритих змін за минулу добу. Це страхувальний механізм на випадок, якщо кур'єр забув завершити зміну.

## 4. Нагадування про X-звіт

Щодня о **12:00**, **16:00** та **20:00** за київським часом бот надсилає у визначений чат повідомлення:

```
Нагадування - час відправити X-звіт через бота.
```

---

## Змінні середовища

| Змінна | Опис |
|---|---|
| `COURIERS` | Список username кур'єрів через кому |
| `VIDEO_CHAT_ID` | ID чату для пересилки відео доставок |
| `PRODUCT_DELIVERED_STATUS` | ID статусу "Доставлено" в CRM |
| `PUBLIC_SHIFT_API_KEY` | API-ключ для SunWays |
| `SUNWAYS_API_BASE_URL` | URL ендпоінту змін (за замовч. `https://sun-ways.vercel.app/api/public/shifts/by-telegram`) |
| `SUNWAYS_VEHICLES_API_URL` | URL ендпоінту авто (за замовч. `https://sun-ways.vercel.app/api/public/vehicles`) |
| `SUNWAYS_AUTO_CLOSE_SHIFTS_URL` | URL крону автозакриття (за замовч. `https://sun-ways.vercel.app/api/cron/auto-close-shifts`) |
| `X_REPORT_REMINDER_CHAT_ID` | ID чату для нагадувань про X-звіт |
